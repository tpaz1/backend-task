# docs/

Working artifacts from the assignment build, kept in the repo for transparency about the AI-assisted workflow used.

- **CLAUDE.md** — the constraint file given to Claude Code: code style, architectural rules, anti-patterns to avoid, and the build workflow. This file is read automatically by Claude Code when working in the repo.
- **FIRST_15_MINUTES.md** — the playbook I followed when the assignment spec landed: read twice, bootstrap a domain skill from the spec, propose 2 design options, plan a walking skeleton, then build.

The `.claude/` directory at the repo root contains the skill files Claude Code loaded during the build (domain vocabulary, abstractions, tradeoffs derived from the assignment spec). It's standard Claude Code project config.

These are not part of the assignment deliverable — they document how the deliverable was produced.
